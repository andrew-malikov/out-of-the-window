# Window Theme — Language Samples

## How to use

1. Open any file in this folder.
2. Switch between Window theme variants.
3. Compare token clarity for comments, keywords, types, functions, strings, numbers, constants, operators, and punctuation.

## Coverage goals per sample

Each sample should intentionally include:

- comments
- declarations and control-flow keywords
- types/classes/interfaces (where applicable)
- functions/methods and calls
- variables/parameters/properties
- strings (single/double/template where available)
- numbers/constants/enums
- operators and punctuation
- one invalid or edge-case token (if grammar supports it)

## Notes

- Keep files short (~30–80 lines each) so previews stay quick and scannable.
- Prefer idiomatic syntax over artificial token dumps.
- Include a compact “real-world” mini example in each file for practical readability checks.
